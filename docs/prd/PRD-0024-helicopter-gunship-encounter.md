# PRD-0024 Helicopter Gunship Encounter

## Vision

`v37` 的目标不是立刻在主世界里凭感觉塞进一架会飞的敌人，然后边跑边补逻辑；而是先建立一条**可独立 `F6` 调试、可复用到主世界任务链、可被正式测试锁定**的直升机炮艇遭遇战主链。玩家先在一个独立 lab 场景里进入任务起始圈，随后空中生成一架精英炮艇；炮艇围绕战区盘旋，持续进行机炮压制与导弹攻击；玩家可以使用现有 `8` 号玩家导弹武器和其他正式武器反击，直到把炮艇击落。等这条链在低干扰环境里调顺之后，再把同一套 runtime 移植回主世界 `chunk_101_178` 的正式任务圈。

`v37` 的关键不是“空中有个模型会动”，也不是“炮艇会发几个粒子看起来像攻击”。它要同时满足六件事。第一，炮艇必须拥有正式 scene/runtime，而不是临时把 `helicopter_a.glb` 当摆设。第二，炮艇必须能在空中盘旋并形成稳定攻击节奏，至少包括持续扫射和无限导弹两种攻击手段。第三，炮艇必须拥有正式生命值 contract，并保证**至少能扛住 10 发玩家导弹直接命中**，不能三两发就掉。第四，主世界接入必须继续沿 `v14` 的 `available -> active -> completed` 任务主链扩展，但本轮完成条件不再是“进 objective 圈”，而是**击落炮艇才完成**。第五，本轮明确**不设计失败态，也不设计玩家掉血**，敌方火力只提供压迫感、爆炸和画面反馈，不引入额外死亡/复活系统。第六，这个任务必须是**可无限重复的遭遇战**：玩家每击落一次炮艇，就算完成一次正式 run；随后 encounter 与任务都回到初始 `available` 状态，玩家再次进入绿圈时可以重新开一局。

## Background

- `PRD-0007` 与 `v14` 已冻结正式任务主链：
  - `task catalog + task slot index -> task runtime -> task pin / task brief / world ring -> route target / start trigger`
- 当前任务 runtime 只原生支持：
  - `available`
  - `active`
  - `completed`
  - 以及“进入 objective slot 即完成”的默认完成方式
- `PRD-0022` 与 `v32` 已冻结玩家导弹武器主链：
  - 正式 weapon mode：`missile_launcher`
  - 正式导弹 scene：`res://city_game/combat/CityMissile.tscn`
  - 当前玩家导弹爆炸伤害为 `14.0`
  - 当前玩家导弹最大飞行距离为 `500m`
- 仓库中已经有独立 lab 先行、主世界后移植的成熟样板：
  - `v33` 的 `BuildingCollapseLab`
- 本轮炮艇视觉资源已经冻结为：
  - `res://city_game/assets/environment/source/aircraft/helicopter_a.glb`
- 用户已经冻结主世界首个正式触发位置：
  - `chunk_id = chunk_101_178`
  - `chunk_key = (101, 178)`
  - `world_anchor = Vector3(-8981.45, 0.0, 10796.22)`
  - `chunk_local_anchor = Vector3(34.55, 0.0, 100.22)`
- 用户已经冻结本轮三条硬边界：
  - 炮艇是“精英炮艇”，具备盘旋扫射 + 间歇导弹
  - 导弹不设弹药量上限
  - 玩家不掉血、任务不失败，只有把炮艇击落才算完成
- 用户还追加冻结了一条 replay 边界：
  - 该任务必须可无限次重复；击落炮艇后回到初始状态，再次进入绿圈即可重新通关

## Scope

本 PRD 只覆盖 `v37 helicopter gunship encounter`。

包含：

- 新增一个可 `F6` 独立运行的炮艇遭遇战 lab 场景
- 新增正式直升机炮艇 scene/runtime，消费 `helicopter_a.glb`
- 炮艇正式状态机、生命值、受击反馈、击落销毁链
- 炮艇空中盘旋逻辑
- 炮艇机炮压制与无限导弹攻击链
- 敌方导弹复用玩家导弹 scene/model 资产思路，不另起一套低质量导弹视觉
- 炮艇至少扛住 `10` 发玩家导弹命中的 survivability contract
- 任务完成条件从“进圈完成”扩展为“事件完成：击落炮艇”
- 任务完成后自动回到初始可接状态，支持无限次重复挑战
- 主世界 `chunk_101_178` 的正式任务圈接入
- world tests / e2e / verification 文档规划

不包含：

- 不做玩家生命值、护甲、倒地或失败惩罚
- 不做任务失败状态、任务重开菜单或死亡回档
- 不做多架炮艇编队
- 不做警察/派系/通缉联动
- 不做真实空气动力学、复杂导弹锁定或弹药系统
- 不做过场动画、语音演出或剧情对话树
- 不做炮艇离场后跨 session 持久化
- 不做“完成后永久从地图消失”的一次性任务口径

## Non-Goals

- `v37` 首版不追求军事模拟精度
- `v37` 首版不追求玩家受击系统；敌方攻击的目标是制造战斗压力和视觉反馈
- `v37` 首版不追求失败态或复杂任务分支
- `v37` 首版不追求主世界多个空战任务共享完整 encounter framework

## Requirements

### REQ-0024-001 必须先交付一个独立的炮艇遭遇战 lab 场景

**动机**：主世界 streaming、任务 UI、导航与战斗同时调试太重，先要有一个 `F6` 即可进入的低干扰实验场，把炮艇盘旋、攻击、受击与击落链完整跑通。

**范围**：

- 新增一个独立 lab 场景，场景内静态 author：
  - 地面
  - 光照 / 环境
  - 玩家
  - combat root
  - 任务起始圈或等价 encounter start volume
  - 炮艇生成/盘旋的空域 anchor
- lab 场景必须尽量复用未来主世界要用的正式 encounter runtime，而不是写一套 lab-only 假逻辑
- lab 场景中进入起始圈后，炮艇必须在空中生成并开始攻击

**非目标**：

- 不要求 lab 场景复刻主世界全部 chunk streaming 细节

**验收口径**：

- `F6` 运行后可进入 lab 场景，并能靠进入起始圈开始遭遇战。
- 玩家可在 lab 场景中用正式武器链攻击炮艇。
- 反作弊条款：不得把 lab 做成脚本里临时 `Node3D.new()` 搭起来的一次性测试壳。

### REQ-0024-002 系统必须提供正式的直升机炮艇 scene/runtime，并消费 `helicopter_a.glb`

**动机**：如果只有一个静态模型或一段动画，就不是正式敌人。

**范围**：

- 新增正式炮艇 scene/runtime，根视觉资产来自：
  - `res://city_game/assets/environment/source/aircraft/helicopter_a.glb`
- 炮艇最小正式状态至少包括：
  - `idle`
  - `spawn_intro`
  - `orbit_attack`
  - `destroyed`
- 炮艇必须拥有正式空中盘旋逻辑，而不是在空中定点悬停不动
- 攻击谱必须至少包含：
  - 机炮/扫射
  - 导弹攻击
- 敌方导弹必须复用正式导弹 scene/model 资产族，不允许再做一套质量明显更差的占位导弹

**非目标**：

- 不要求 `v37` 首版加入复杂躲避、锁定、热焰弹或导弹库存

**验收口径**：

- 自动化测试至少断言：炮艇 scene 可加载，并保留正式视觉资产路径。
- 自动化测试至少断言：炮艇在若干 physics frame 后会维持空中 orbit，而不是落地或卡在原点。
- 自动化测试至少断言：炮艇会调度机炮与导弹两条攻击链。
- 反作弊条款：不得只播放 rotor 动画或沿一条死轨迹飞一圈就宣称“敌人逻辑已完成”。

### REQ-0024-003 炮艇必须具备正式生命值与击落 contract，且至少扛住 10 发玩家导弹

**动机**：用户已经明确要求“起码要能扛住我的 10 发导弹攻击”，而且任务完成条件只能是击落炮艇。

**范围**：

- 炮艇必须维护正式生命值 state，至少包含：
  - `max_health`
  - `current_health`
  - `alive`
  - `destroyed`
  - `last_hit_world_position`
- 当前玩家导弹伤害是 `14.0`；在该口径下，炮艇首版应以“至少承受 10 发直接玩家导弹命中仍未被击毁”为硬约束
- 推荐首版用 `max_health = 160` 作为基线，但正式验收以“10 发 direct missile hit 不死”为准，不只看裸数值
- 任务只有在炮艇被击落后才允许完成

**非目标**：

- 不要求首版加入部位伤害、螺旋桨断裂、尾翼失效等细粒度系统

**验收口径**：

- 自动化测试至少断言：炮艇在连续 10 发玩家导弹直接命中后仍存活。
- 自动化测试至少断言：继续追加伤害后炮艇能进入 `destroyed` 状态。
- 自动化测试至少断言：炮艇销毁事件会被 encounter / task runtime 消费为正式完成信号。
- 反作弊条款：不得通过把前 10 发导弹判成 miss、关闭碰撞或临时禁伤来伪造 survivability contract。

### REQ-0024-004 主世界任务接入必须继续复用 `v14` 主链，但完成条件要扩展为事件完成

**动机**：当前 `v14` 默认是“进 start 圈开始，进 objective 圈完成”；本轮需要的是“进 start 圈开始，击落炮艇完成”，但不能因此再造第二套任务系统。

**范围**：

- 正式任务状态仍冻结为：
  - `available`
  - `active`
  - `completed`
- 本轮不引入 `failed`
- 本轮允许在 `completed` 之后按 repeatable contract 自动回到 `available`
- `chunk_101_178` 的正式起始圈固定在：
  - `Vector3(-8981.45, 0.0, 10796.22)`
- active objective 仍要继续产出正式 route / pin / world ring 语义
- 但完成方式从“enter objective slot”扩展为“event objective completion”
- 炮艇被击落后，由 encounter runtime 向 `task runtime` 发送正式完成事件
- 对于本任务这种 repeatable encounter，完成事件 closeout 后必须恢复 start ring 和 route 起点，让玩家下一次重新进入绿圈时开启新 run

**非目标**：

- 不要求 `v37` 把整个任务系统一次性重写成任意脚本型 objective graph

**验收口径**：

- 自动化测试至少断言：进入 `chunk_101_178` 的 start ring 后，任务能从 `available` 进入 `active`。
- 自动化测试至少断言：active 后 shared route/world ring 仍然来自同一 task runtime 主链。
- 自动化测试至少断言：击落炮艇会让任务进入 `completed`，且不依赖进入第二个完成圈。
- 自动化测试至少断言：完成 closeout 后，该任务会恢复为 `available`，且必须重新进入绿圈而不是原地连触发，才能开启下一次 run。
- 反作弊条款：不得通过单独做一套 gunship-only 状态机、绕开 `task runtime` 来宣称主世界任务已接入。

### REQ-0024-005 敌方火力必须成立，但本轮不产生玩家掉血或失败态

**动机**：用户已经明确冻结本轮“不设计过于复杂”，玩家不掉血、任务不失败。

**范围**：

- 炮艇机炮与导弹都必须真实发射、飞行、命中地面或空域并产生攻击反馈
- 导弹数量不设上限
- 机炮/导弹可以制造：
  - tracer / 爆炸 / 近失反馈
  - camera shake
  - 地面冲击或视觉压力
- 本轮不得引入：
  - 玩家生命值扣减
  - 任务失败
  - 离区失败
  - 死亡重开

**非目标**：

- 不要求这轮构建完整 player damage / revive / checkpoint 系统

**验收口径**：

- 自动化测试至少断言：炮艇会持续调度敌方火力，而不是只在 UI 上标注“正在攻击”。
- 自动化测试至少断言：玩家不会因为炮艇攻击而触发失败态或 health mutation。
- 反作弊条款：不得把“不掉血”偷换成“敌人根本不攻击”。

### REQ-0024-006 该任务必须支持无限次重复通关，并在每次击落后回到初始状态

**动机**：用户已经明确要求这不是一次性剧情任务，而是可反复游玩的遭遇战内容。

**范围**：

- 炮艇被击落后，当前 run 记为一次正式完成
- encounter closeout 结束后，系统必须恢复到初始状态：
  - 炮艇不存在
  - encounter runtime 复位
  - 绿圈重新可见
  - 任务重新回到 `available`
- 下一次 run 必须要求玩家重新进入绿圈触发，而不是停在原地自动连续刷怪
- runtime 应保留最小可观测重复指标，例如：
  - `completion_count`
  - 或等价 `successful_run_count`

**非目标**：

- 不要求首版把重复通关历史持久化到跨 session 存档
- 不要求首版为每次通关生成奖励或排行榜

**验收口径**：

- 自动化测试至少断言：第一次击落炮艇后，encounter 会回到初始可接状态。
- 自动化测试至少断言：玩家再次进入绿圈会重新生成新一架炮艇并开始第二次 run。
- 自动化测试至少断言：重复通关计数或等价 run 统计会增加。
- 反作弊条款：不得通过重新加载整个场景或重开游戏来冒充“任务可重复”。

## Success Metrics

- 独立 lab 中进入起始圈后，炮艇能正式生成、盘旋并攻击
- 玩家可用现有 `8` 号导弹等正式武器反击
- 炮艇至少扛住 10 发玩家导弹直接命中
- 主世界 `chunk_101_178` 的任务圈可启动同一 encounter，且击落炮艇后正式完成任务
- 击落后 encounter 能回到初始状态，再次进入绿圈可重新开始下一轮
